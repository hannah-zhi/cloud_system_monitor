# 储能云端安全预警系统 PRD

## 1. 产品定位

“安全预警”用于集团级管理 100+ 储能场站的安全态势、SOS 风险等级和预警闭环入口。页面以首页场站矩阵为主视图，右侧提供预警清单；点击场站或预警进入单站安全诊断详情。

## 2. 目标用户

- 集团安全运营管理员：查看全集团场站风险分布和高严重度预警。
- 区域运维负责人：按场站、风险等级筛选并定位预警位置。
- 单站值班人员：进入单站详情查看 SOS 趋势、子系统风险和预警明细。

## 3. 页面结构

安全预警采用二级菜单结构：

- 一级菜单：安全预警。
- 二级菜单：安全监测、安全评估、预警管理、风险运营。

其中前三个二级菜单分别对应现有三张主页面：

- 安全监测：当前主页面，展示场站矩阵和右侧预警清单。
- 安全评估：面向全集团场站 SOS 情况的图表看板。
- 预警管理：预警清单的独立明细页，支持复杂筛选和预警详情查看。
- 风险运营：当前保留为导航入口位，未承载独立页面内容。

## 4. 场站总览

- 左侧导航：远景数智运营系统；菜单包含全景概览、安全预警、风险运营、健康管理、运营评估、交易协同、数据分析、数据运营。
- 场站筛选：搜索/下拉多选场站，支持恢复全部场站。
- 场站总览顶部工具栏与侧边栏之间需保留清晰边距，搜索场站框不得贴近或重叠侧边栏分割线。
- 排序：场站序号、SOS 指数。
- 风险筛选：高风险、中风险、低风险、健康。
- 场站卡片字段：通讯状态、额定容量 MW、额定能量 MWh、子系统数量、场站类型、SOS 进度条。
- 预警清单：固定全量 273 条预警，按左侧当前场站筛选结果同步过滤展示。
- 预警清单右上角提供“查看详情”按钮，可直接跳转到“预警详情”独立页面；点击场站总览或单站详情中的任意预警卡片时，均以“预警详情”页同款弹窗展示完整预警信息、处置建议和相关数据趋势，弹窗层级必须高于场站筛选栏、下拉框等所有页面控件。

## 5. 风险透视

- 顶部第一行展示风险场站 TOP5、全量场站平均 SOS、场站风险等级占比三个模块。
- 风险场站 TOP5 按 SOS 从低到高排列，每行展示场站名称、SOS 和风险色进度条。
- 全量场站平均 SOS 独立成仪表盘卡片，固定按全部有效场站计算，不随当前筛选条件变化；仪表盘颜色从红、黄到绿平滑渐变，不展示“当前风险等级”文字，刻度密度需与数字比例协调。
- 场站风险等级占比环图展示高风险、中风险、低风险、健康数量，随筛选变化；legend 与图形保持居中紧凑排布。
- SOS 透视区：
  - 场站SOS分布与全量场站SOS趋势放在同一行，场站SOS分布在左侧，全量场站SOS趋势在右侧。
  - 全量场站SOS趋势图为点线图，展示全量场站平均值、最大值、最小值，横坐标显示日期，支持 7天、15天、30天快捷窗口。
  - 日期统一使用 `MM-DD` 格式，例如 `04-01`。
  - 趋势图阈值线与场站SOS分布保持一致：`80` 为黄色虚线，`60` 为红色虚线。
  - 鼠标悬停趋势点或日期列时显示垂直对齐线，并同时展示最大值、平均值、最小值三个 SOS 数值；同一日期的三个点同步高亮；最小值曲线使用紫色。
  - 场站SOS分布柱状图，细柱样式，颜色按风险等级区分。
  - 场站SOS分布支持排序：场站编号-顺序、场站编号-倒序、SOS数值-从低到高、SOS数值-从高到低。
  - 柱状图支持固定可见的横向滚动条，横坐标显示场站号。
  - 横坐标区域不展示缩略曲线，仅保留场站号刻度。
  - 鼠标悬停柱子时展示完整场站名称和 SOS 数值，SOS 数值颜色与柱子风险色一致，并以单柱宽度附近的半透明竖向背景高亮当前柱子。
- 预警透视区：
  - 预警等级占比：以饼图统计全部、一级、二级、三级中的等级构成。
  - 预警模块分布：以饼图统计电池系统、电气系统、环控系统、消防系统。
- 预警类型 TOP5：按预警名称/类型聚合数量，使用进度条列表展示，数量需有差异；配色依次为红、黄、绿、青、紫。
- 图表数据随左侧场站筛选条件同步变化。

## 6. 预警详情

- 页面采用顶部横向筛选条 + 全宽预警列表布局，参考风险预警列表样式。
- 顶部 3 张预警透视图需与下方表格共用同一套筛选条件，随时间、等级、模块、预警名称、场站、位置、来源筛选结果同步变化。
- 预警类型 TOP5 的排序、进度条长度与右侧数字均基于当前筛选结果的真实聚合计数，不得叠加演示用修饰数值。
- 默认全量场景下，预警类型 TOP5 需呈现递减分布，不应出现前 5 名数量完全相同的平均化结果。
- 查询栏包含时间、等级、状态、模块、预警名称、场站、位置、来源筛选；“状态”筛选框位于“等级”后方，并需比等级筛选框更宽。
- 等级、模块、来源支持多选下拉；预警名称、场站、位置支持可搜索的多选下拉。
- 预警详情页多选下拉框与场站总览“搜索场站”下拉框保持同一组件样式，包括触发框、箭头、搜索框、选项行、勾选态、阴影和图层优先级；后续全系统多选/搜索下拉组件统一复用该样式。
- 时间筛选采用开始日期、结束日期范围选择。
- 表格字段：等级、预警名称、模块、场站、位置、事件时间、预警时间、来源。
- 表格需按 `预警名称 + 模块 + 场站 + 位置 + 来源` 聚合同类预警，同一组合只展示一行；云端与站端来源不得合并到同一行或同一详情卡片内；等级、事件时间、预警时间、状态均取该组合中最新一条预警的数据。
- 表格新增“状态”列，状态流转规则：
  - 初始生成：`待处理`。
  - 首次处理选择关闭：`关闭-误报` / `关闭-数据异常` / `关闭-其他`。
  - 首次处理选择下发 SR：`排查中`。
  - SR 完成返回后再次关闭：`关闭-准确` / `关闭-类型不准确`；如所选失效模式仍需根因补充，则先进入 `关闭-待补充根因`，根因提交完成后再变更为最终关闭状态。
- 所有 `关闭-XXX` 状态标签统一使用灰色样式。
- 一部分来源为 `站端` 的预警初始状态为 `关闭-站端已处理`；如该站端预警存在关联云端预警，则关联云端预警自动同步为 `关闭-站端已处理`。
- 下发 SR 并完成返回后，表格在“等级”列之前展示信封标记，提示该聚合预警存在 SR 回执。
- 预警管理表格列宽策略：压缩“等级、模块、来源”列宽，优先保证“预警名称、场站、位置”三列的展示空间，事件时间与预警时间列保持稳定宽度，不随其它列调整而被过度挤压。
- 点击任意聚合预警行后，以弹窗卡片形式展示该组合下多条不同时间的预警；上方使用行式列表展示同组预警，最新预警在最上方并按预警时间倒序排列，点击不同行后下方相关数据趋势图随之切换，不在页面底部固定展示详情。
- 同组预警行上方单独展示模块、场站、位置三个信息框；预警行字段顺序为：序号、等级、事件时间、预警时间、关闭时间、持续时长、状态；序号和等级列需适当加宽并居中对齐，持续时长列需缩窄，事件时间、预警时间和关闭时间列需加宽，序号直接展示 `1/2/3/4...`，不展示“来源”和“位置”列，避免列间重叠。关闭时间在预警关闭时自动填充，未关闭时留空。前几条风险预警列表卡片需提供多条不同时间、不同等级的同组预警行，点击不同行后下方趋势图切换。
- 预警详情弹窗突出展示预警名称和处置建议；预警来源保持顶部主卡左上角三角形式，文字方向斜 45 度，文字本身采用类似等级标签的小型描边 pill 风格，云端为蓝色系、站端为绿色系；顶部主卡不再展示“一级”“电池系统”等标签。如存在关联的云端/站端告警，顶部主卡右下角展示关联跳转按钮，按钮颜色与目标关联告警来源匹配：云端为蓝色，站端为绿色。详情卡片下方不再展示独立的来源、事件时间、预警时间、持续时长、当前状态信息框。
- 预警来源三角只保留三角背景和外框线，不在三角内额外叠加方框；云端/站端文字字号需略大于此前版本以保证可读性。
- 预警详情弹窗顶部底色需匹配预警等级：一级为红蓝渐变，二级为黄蓝渐变，三级为绿蓝渐变。
- 预警详情页所有完整时间显示到秒，格式为 `YYYY/MM/DD HH:mm:ss`。
- 弹窗包含可交互趋势曲线，用于展示该预警相关数据在事件前后的变化；曲线下方提供“处理”和“分析”跳转按钮。
- 点击“处理”后，所有后续处理内容必须在新的居中弹窗中展示，不在详情弹窗底部展开，避免用户需要下拉才能看到处理步骤。
- 点击“处理”后的流程：
  - 第一步选择“关闭预警”或“下发 SR”。
  - 处理方式弹窗中不展示“请选择本组预警闭环路径”等说明小字。
  - 选择“关闭预警”时，需进一步选择 `误报`、`数据异常`、`其他`；选择 `其他` 时必须填写具体原因。
  - 选择“下发 SR”时展示 SR 表单，弹窗最上方固定展示 `场站`、`位置`、`模块`、`预警名称` 四个上下文字段；表单第一行为 `SR编号`，第二行为 `操作指导`，第三行为附件，第四行为 `SR下发时间` 与 `期望完成时间`；不展示“工单编号”；两个时间字段均为日期+时分格式，`SR下发时间` 默认为打开“下发 SR”时的北京时间；“操作指导”默认空白。
  - SR 下发后预警状态进入 `排查中`；待 SR 完成返回后，聚合预警行出现信封标记，再次进入预警卡片可选择关闭。SR 返回确认弹窗第一行展示 `SR编号` 与 `关联工单编号`，第二行为只读回传的确定性 `排查结论`，例如“现场已完成排查，确认该Pack存在铜排螺栓松动”，第三行为 `类型准确` / `类型不准确` 确认项，最下方为预警相关失效模式下拉框；下拉框包含若干与预警名称关联的失效模式和 `其他`，选择 `其他` 时必须手动输入具体失效模式。
- 对于部分仍需进一步归因的失效模式，点击确认关闭后状态先变为 `关闭-待补充根因`，并在预警卡片下方展示根因填写框；只有根因填写完成并提交后，对应状态才变为 `关闭-准确` 或 `关闭-类型不准确`。
- 已完成 `类型准确/类型不准确` 和失效模式选择后，再次点击“处理”时不再重复展示选择表单，而是直接展示已选择的 SR 返回结果；只有状态为 `关闭-待补充根因` 时，结果弹窗底部展示根因补充输入。
- 对于 `关闭-站端已处理` 的预警，详情卡片在闭环摘要内展示处理动作与排查结论，并禁用“处理”按钮，无需再次处理。
- 预警详情页数据来源与场站总览右侧预警清单一致，均从固定 273 条全量预警中按当前场站范围过滤。

## 7. 预警清单规则

- 顶部标签：全部、一级、二级、三级；数字随当前场站筛选和时间筛选变化。
- 等级分布：全量 273 条中一级 35、二级 70、三级 168。
- 来源分布：云端多于站端，当前规则约为 3:1。
- 来源颜色：云端蓝色，站端绿色。
- 时间：显示 `MM-DD HH:mm`，不显示 `(+08:00)`。
- 场站名称格式：与场站卡片保持一致，如 `K-0005远景乌兰察布电网侧储能`。
- 时间筛选：默认全部；快捷按钮为全部、最近30天、最近7天、最近3天；支持手动日期范围。
- 预警内容来源：`risk_list_demo.xlsx`，字段包括预警名称、模块、风险等级、位置格式。
- 预警位置：鼠标悬停或聚焦预警卡片时展示。

## 8. 数据规则

- 场站基础数据来自 `site_config.xlsx`，有效场站需同时具备项目号、项目名称、额定容量、额定能量、总套数。
- 多行场站按项目号聚合，总套数累加。
- 场站名称中的英文括号统一替换为中文括号。
- 已剔除信息不完整的场站：`待定`、`K-北方片区`。
- 当前有效场站总数为 158。
- 场站类型按比例生成：配套储能约 85%，独立储能约 10%，工商业储能约 5%。

## 9. SOS 风险规则

- SOS < 60：高风险。
- 60 <= SOS < 80：中风险。
- 80 <= SOS < 100：低风险。
- SOS = 100：健康。
- 首页目标分布：健康 21 个，其余数量向低风险和中风险倾斜。
- SOS 展示：100 显示为 `100`；其他值保留两位小数。

## 10. 预警位置生成规则

- 子系统编号范围：`1 - 场站总套数`。
- Rack：形如 `101` 或 `201`，首位为 1 或 2，后两位范围 01-12。
- Pack：1-8。
- Cell：1-28。
- 非电池系统位置保持模板结构，例如 `LCC01`、`BankA-TCQ01`、`AC101`，仅替换子系统编号。

## 11. 单站详情

- 从场站卡片或预警卡片点击进入。
- 单站详情页顶部不展示“集团安全中心 / 安全预警 / 场站名称”面包屑。
- 顶部右侧保留通讯状态、风险等级、SOS 标签，风险等级和 SOS 数值颜色需按照高风险红色、中风险黄色、低风险绿色、健康蓝色匹配。
- 顶部额定功率、有功功率、剩余电量、SOC、当前告警指标条不展示。
- 页面主体采用左图表 + 右预警清单布局；左侧为 SOS 仪表盘、历史趋势、风险等级分布、子系统 SOS、设备部件箱线图和详情表格；右侧为当前场站预警清单，样式与场站总览页右侧预警清单保持一致，并支持等级切换、时间快捷筛选、自定义日期范围和来源统计联动。
- 右侧预警清单底部需与左侧 SOS 安全指数详情表格底部对齐。
- 单站右侧预警清单卡片不再悬停展示“预警位置”浮层。
- 单站详情页所有仪表盘、饼图、柱状图、曲线图和表格的深色科技风、风险配色、面板线框、标题样式、标签样式需与风险透视和预警详情保持一致。
- 单站详情页 SOS 仪表盘使用与风险透视“全量场站平均 SOS”一致的刻度式半圆仪表盘，不使用块状扇形仪表盘。
- 单站详情页 SOS 仪表盘数值使用固定浅色，不随风险等级变化；数值需位于半圆内部的视觉中心。
- 单站详情页曲线图、柱状图、箱线图需支持与风险透视一致的悬停交互：当前点/柱高亮，并展示对应日期、子系统、SOS 或统计参数。
- 单站详情页子系统 SOS 柱状图使用细柱样式，柱间距需大于柱宽，避免视觉拥挤。
- 单站详情页历史趋势图不显示“高风险/中风险”阈值文字，也不显示 y 轴标题；x 轴日期始终使用 `MM-DD` 格式，窗口变长时仅降低标签密度，不改变格式。
- 单站详情页子系统 SOS 柱状图右上角使用下拉排序，选项包括子系统编号-顺序、子系统编号-倒序、SOS数值-从低到高、SOS数值-从高到低；同时展示高风险、中风险、低风险、健康 legend。
- 单站详情页子系统 SOS 柱状图 x 轴标签使用 `#01`、`#02` 这类带 `#` 的两位编号格式。
- “设备部件 SOS 指数分析”为箱线图，包含上边缘、上四分位、中位数、均值、下四分位、下边缘和离群点；悬停箱体展示关键统计参数，悬停离群点展示具体子系统信息。
- “设备部件 SOS 指数分析”箱线图各箱体之间需保留清晰间距，避免贴得过近。

## 11.1 2026-05-08 预警管理补充规则

- 已关闭预警在详情弹窗首页直接展示闭环摘要，包括 `SR编号`、`操作指导`、`排查结论`、`预警准确/不准确`、`失效类型`；仅当状态为 `关闭-待补充根因` 时继续展示根因补充输入框，不需要补充根因的已关闭预警再次点击处理时不展示根因输入。
- 根因补充完成后，根因内容需并入详情弹窗首页闭环摘要；无需补充根因的预警在处理结果弹窗中不展示“根因/无需补充根因”字段。
- 根因摘要在闭环摘要区内独占整行展示，避免短卡片导致内容拥挤。
- 对于存在云端/站端关联关系的预警，任意一侧发生处理状态、SR 字段、排查结论、预警准确性、失效类型、根因等闭环字段变更时，另一侧关联预警需同步更新。

- 已下发 SR 且状态为 `排查中`、但 SR 尚未返回时，再次点击“处理”应直接展示“模拟 SR 完成返回”入口，不再展示“关闭预警 / 下发 SR”的第一步选择。
- `K-0005` 场站前两条同组预警保留为可演示处理流程，不默认设置为 `关闭-站端已处理`。
- 预警详情筛选栏中 `状态` 筛选框位于 `等级` 后方，触发框保持紧凑，下拉面板可宽于触发框并完整承载状态文案；筛选栏不得出现横向滚动条，且下拉面板不得被表格表头遮挡。
- 风险预警列表中的 `等级` 表头与等级标签必须在同一列中水平居中对齐，普通浏览模式不保留选择框占位。
- 预警详情卡片内的事件时间、预警时间、关闭时间三列适当减小列宽，保证持续时长和状态列可见且不重叠。

## 12. 视觉规范摘要

- 深色科技风，参考安全诊断页面。
- 重点色：蓝色用于导航和主强调；红/黄/绿用于风险等级；绿色用于健康和站端来源。
- 卡片边框和面板使用低透明度线框、微渐变背景、左侧蓝色强调线。
- 主体验不使用浮夸装饰，强调信息密度、层级和稳定布局。
